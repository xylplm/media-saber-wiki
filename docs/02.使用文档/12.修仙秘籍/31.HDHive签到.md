# 🐝 HDHive 签到

**HDHive 签到** 插件用于自动登录并完成 HDHive 站点的每日签到任务，避免漏签。

## ⚙️ 功能特点

- 自动完成 HDHive 每日签到
- 支持定时执行签到任务
- 可选开启消息推送，查看签到结果

## 🚀 配置指南

插件配置字段与 HDHive 登录方式直接相关，按界面提示填写即可：

### 📋 基础配置

1. **用户名 (username)**
   - 必填；你的 HDHive 登录用户名；
   - 建议与网页端登录保持一致，避免填错大小写或带多余空格。

2. **密码 (password)**
   - 必填；你的 HDHive 登录密码；
   - 如启用了二步验证或额外验证方式，以站点实际要求为准。

3. **周期 (cron)**
   - 选填；使用 Cron 表达式设置签到执行时间；
   - 示例：`0 8 * * *` 表示每天早上 8 点自动签到；
   - 若留空，则不会自动运行，只能通过「运行一次」手动执行。

4. **消息推送 (notice)**
   - 选填；开启后，每次签到都会推送一条消息；
   - 内容通常包括：签到是否成功、错误原因等，便于你快速确认状态。

### 🔧 工具栏操作

- **运行一次 (run)**：立即执行一次签到任务，用于测试配置是否正确。

## 📖 使用说明

1. 在浏览器中确认可以正常登录 HDHive，确保账号状态正常；
2. 在本插件配置中填写与网页一致的用户名与密码；
3. 先不设置 `cron`，直接点击「运行一次」做一次手动测试：
   - 成功时，会提示今日签到成功或已签到；
   - 失败时，请检查用户名/密码是否正确，或是否存在验证码等额外验证；
4. 测试通过后，再设置每天固定时间的 `cron` 表达式，交给系统自动执行；
5. 如希望随时收到签到结果，开启「消息推送」，并在消息通道中勾选本插件。

## ⚠️ 注意事项

- 不要设置过于频繁的签到周期，每天一次即可；
- 连续多次失败可能会触发站点风控，建议先暂停任务排查原因；
- 若站点登录方式或签到入口有调整，请关注插件更新或社区公告；
- 如你启用了多重验证（如二步验证），本插件可能无法完成登录，此时建议改用站点支持的其他自动化方式。

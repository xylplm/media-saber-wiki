# 🍪 CookieCloud 同步

**CookieCloud 同步** 插件用于从 CookieCloud 服务器同步站点的 Cookie，支持自动新增或更新站点 Cookie，方便在多设备、多环境之间共享登录状态。

## ⚙️ 功能特点

- 从 CookieCloud 服务端拉取最新 Cookie 数据
- 自动匹配 Media Saber 中已有站点并更新 Cookie
- 可选自动新增站点条目
- 支持定时同步与手动「立即同步」
- 支持域名黑名单过滤不需要同步的站点

## 🚀 配置指南

在插件配置页面中可以看到所有字段，下面按界面顺序说明每一项如何填写：

### 📋 必填参数

1. **服务地址 (server)**
   - 必填
   - CookieCloud 服务器的访问地址
   - 示例：`http://localhost:8088` 或 `https://your-domain.com`

2. **用户 Key (userKey)**
   - 必填
   - 在 CookieCloud 客户端中生成/查看的用户标识

3. **端对端加密密码 (encryptKey)**
   - 必填
   - 在 CookieCloud 客户端中设置的「端对端加密」密码；
   - 必须与浏览器侧/客户端中填写的一致，否则无法解密远端数据。

### ⚙️ 可选参数

4. **执行周期 (cron)**
   - 选填
   - 使用 Cron 表达式设置定时同步周期
   - 默认示例：`0 2 * * *` （每天凌晨 2 点执行）

5. **通知 (notice)**
   - 选填
   - 开启后，每次同步完成会发送一条包含成功 / 失败数量等统计信息的通知

6. **自动增加站点 (autoAddSite)**
   - 选填，默认可按实际表单为准；
   - 开启：当 Cookie 数据中包含的站点在 Media Saber 中还不存在时，会自动创建站点条目并写入 Cookie；
   - 关闭：只会更新当前已经存在的站点，不会自动新建，适合希望严格手动管理站点列表的用户。

7. **域名黑名单 (domainBlacklist)**
    - 选填；每行一个域名/后缀，用于排除不希望同步的站点；
    - 支持精确匹配和类似 `.example.com` 这样的后缀匹配；
    - 示例：
       ```
       example.com        # 精确匹配 example.com
       .tracker.com       # 匹配所有以 .tracker.com 结尾的域名
       ```

### 🔧 工具栏操作

- **立即同步 (run)**：立刻发起一次 Cookie 同步，无视 `cron` 设置。

## 📖 使用说明

1. 在浏览器或桌面端先配置并验证好 CookieCloud：
   - 能看到浏览器端 Cookie 正常同步到 CookieCloud；
   - 如开启了端对端加密，记住加密密码。
2. 在本插件中填写 `服务地址`、`用户 Key` 和 `端对端加密密码`（如有）；
3. 根据自己的站点管理习惯，决定是否开启「自动增加站点」：
   - 若希望“一处管理，处处生效”，可以开启；
   - 若希望手动控制站点列表，则关闭。
4. 根据需要将不想同步的站点域名加入「域名黑名单」，例如公共 tracker、临时站点等；
5. 先不设置 `执行周期 (cron)`，点击「立即同步」执行一次测试：
   - 成功时，可在站点配置中看到 Cookie 被更新，通知中也会有成功/失败数量统计；
   - 若失败，多数是因为服务地址/用户 Key/加密密码不匹配，请逐一排查；
6. 测试通过后，再设置合适的 `执行周期 (cron)`，让系统定期自动同步。

## ⚠️ 注意事项

- 仅同步 Media Saber 已支持的站点类型，其他 Cookie 会被忽略。
- 域名黑名单中的站点将完全跳过，不参与同步。
- 请确保 CookieCloud 服务处于可访问状态，且网络连通正常。
- 若端对端加密密码填写错误，将导致数据解密失败，无法同步。
